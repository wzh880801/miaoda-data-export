# 妙搭数据导出助手 v2.0

一个 Chrome 浏览器插件，自动捕获妙搭数据管理后台的请求，一键导出完整数据表，支持人员字段展开导出。

## ✨ 功能特点

- 🎯 **自动捕获请求**：使用 `webRequest` API 自动捕获页面请求，无需手动复制 Cookie
- 📊 **智能导出策略**：
  - **小表**（≤ 5万条）：一次性加载到内存后导出
  - **大表**（> 5万条）：流式导出，逐页写入文件，避免内存溢出
- 👤 **人员字段展开**：同时导出 ID、姓名、邮箱
- 📋 **数组字段处理**：使用分号连接导出
- 💾 **CSV 格式**：导出为 CSV 格式，带 UTF-8 BOM，Excel 直接打开无乱码
- 🎨 **侧边栏模式**：固定在浏览器侧边，流式导出时不会中断
- 📱 **清晰界面**：12px 以上字号，实时显示请求捕获状态和导出进度

## 🚀 安装方法

1. **下载插件**
   ```bash
   # 克隆或下载本项目
   git clone <repo-url>
   ```

2. **加载到 Chrome**
   - 打开 Chrome 浏览器，访问 `chrome://extensions/`
   - 开启右上角的「开发者模式」
   - 点击「加载已解压的扩展程序」
   - 选择 `miaoda-data-exporter` 文件夹

3. **刷新妙搭页面**
   - 在妙搭数据管理后台页面刷新一次
   - 插件会自动捕获请求

## 📖 使用方法

### 自动模式（推荐）

1. **打开妙搭页面**
   - 在浏览器中访问妙搭数据管理后台
   - 确保已登录

2. **刷新页面或切换数据表**
   - 刷新页面或点击不同的数据表
   - 插件会自动捕获请求

3. **打开插件侧边栏**
   - 点击浏览器工具栏的插件图标，侧边栏会自动打开
   - 等待状态显示「✅ 已捕获请求」
   - 侧边栏在导出过程中保持打开，避免中断

4. **获取表列表**
   - 点击「获取表列表」按钮
   - 选择要导出的数据表

5. **导出数据**
   - 勾选/取消「展开人员字段」选项
   - 点击「导出数据」按钮
   - **小表**（≤ 5万条）：自动下载 CSV 文件
   - **大表**（> 5万条）：弹出文件选择器，选择保存位置后流式导出

### 导出模式说明

#### 小表导出（≤ 5万条）
- 数据一次性加载到内存
- 自动生成并下载 CSV 文件
- 进度显示：`自动翻页获取数据中，2/10...`

#### 大表导出（> 5万条）
- 使用流式导出，避免内存溢出
- 弹出文件选择器让用户选择保存位置
- 逐页获取数据并实时写入文件
- 进度显示：`流式导出中，12/500 页 (1200/50000 条)...`

### 手动模式（备用）

如果自动捕获失败，可以手动填写工作区 ID 和应用 ID：

1. 从 URL 中获取：
   ```
   https://miaoda.feishu.cn/app/{appId}/...
   ```
   - **应用 ID**: `app_4jcs92659dk7p`
   - **工作区 ID**: 从 `listTableView` 请求 URL 中获取

2. 在插件中手动填写后点击「获取表列表」

## 🔧 工作原理

### 请求捕获流程

```
┌─────────────┐     ┌─────────────┐     ┌─────────────┐
│  妙搭页面   │────▶│  Chrome API │────▶│  Background │
│  发起请求   │     │ webRequest  │     │   存储请求   │
└─────────────┘     └─────────────┘     └─────────────┘
                                               │
                                               ▼
                                        ┌─────────────┐
                                        │    Popup    │
                                        │  获取请求头  │
                                        │  执行导出    │
                                        └─────────────┘
```

### 捕获的请求类型

1. **`listTableView`** - 获取表列表请求
   - 提取工作区 ID 和应用 ID
   - 获取完整的请求头（包含 CSRF token）

2. **`admin/data/{table}`** - 获取表数据请求
   - 复用请求头进行分页获取

## 🛠️ 技术架构

```
miaoda-data-exporter/
├── manifest.json       # 插件配置（v3，含 webRequest、sidePanel 权限）
├── sidebar.html        # 侧边栏界面
├── sidebar.js          # 侧边栏逻辑（轮询捕获状态）
├── background.js       # 后台服务（webRequest 捕获、侧边栏控制）
├── content.js          # 内容脚本（辅助触发）
└── README.md           # 说明文档
```

### 核心 API

- **`chrome.webRequest.onBeforeRequest`** - 捕获请求 URL 和类型
- **`chrome.webRequest.onBeforeSendHeaders`** - 捕获完整请求头
- **`chrome.sidePanel`** - 侧边栏管理
- **`chrome.runtime.sendMessage`** - Background 与 Sidebar 通信

## ⚠️ 注意事项

1. **首次使用需要刷新页面**
   - 插件需要捕获页面的真实请求才能获取认证信息
   - 刷新页面或切换数据表可以触发请求捕获

2. **保持侧边栏打开**
   - 导出过程中不要关闭插件侧边栏
   - 侧边栏模式确保流式导出过程不会中断

3. **数据量限制**
   - 单次最多导出 10000 条记录
   - 分页获取时有延迟避免请求过快

4. **权限说明**
   - 插件使用 `webRequest` 权限捕获请求
   - 仅在 `miaoda.feishu.cn` 域名下生效
   - 所有数据处理在本地完成，不会上传到任何服务器

## 🐛 故障排除

### 状态一直显示「等待捕获请求」

**原因**：插件没有捕获到页面请求

**解决方法**：
1. 确保在妙搭页面（URL 包含 `miaoda.feishu.cn`）
2. 刷新页面或点击不同的数据表
3. 等待几秒后再次打开插件

### 获取表列表失败

**原因**：捕获的请求已过期或无效

**解决方法**：
1. 刷新妙搭页面
2. 等待状态变为绿色「已捕获请求」
3. 重新点击「获取表列表」

### 导出数据失败

**原因**：请求头过期或权限不足

**解决方法**：
1. 刷新妙搭页面
2. 重新捕获请求
3. 检查是否有该数据表的访问权限
